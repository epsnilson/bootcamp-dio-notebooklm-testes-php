## Como utilizar o NotebookLM para aprender Testes de Software com PHP

## Contexto

Este projeto foi desenvolvido como parte do Bootcamp da DIO com o objetivo de explorar o **NotebookLM** como ferramenta de aprendizagem ativa.

Para isso, escolhi estudar **Testes de Software com PHP**, dividindo o tema em duas etapas:

* **Parte 1:** Testes em PHP sem framework, utilizando PHPUnit.
* **Parte 2:** Testes em aplicações desenvolvidas com Laravel.

A proposta deste projeto não é criar um curso sobre testes de software, mas documentar o processo de aprendizagem utilizando o NotebookLM. Durante o estudo, foram selecionadas fontes confiáveis, elaborados diferentes prompts, refinadas perguntas e consolidados os principais conceitos aprendidos.

Ao final, este repositório reúne um pequeno guia de estudos, um glossário e um conjunto de prompts reutilizáveis que poderão servir como apoio em futuras revisões sobre o tema.

---

## Objetivos

### Objetivo Geral

Utilizar o NotebookLM como ferramenta de aprendizagem para compreender os principais conceitos de Testes de Software com PHP e registrar o processo de estudo realizado durante o Bootcamp.

### Objetivos Específicos

* Explorar o NotebookLM como apoio à organização e compreensão de conteúdos técnicos.
* Estudar os fundamentos dos testes de software aplicados ao desenvolvimento em PHP.
* Conhecer os principais recursos do PHPUnit para criação de testes automatizados.
* Compreender como o Laravel organiza e facilita a implementação de testes.
* Registrar a evolução dos prompts utilizados durante o estudo.
* Consolidar o aprendizado por meio de um miniguia, um glossário e uma coleção de prompts reutilizáveis.

## Curadoria de Fontes

Para construir este caderno temático, optei por utilizar **fontes oficiais e reconhecidas pela comunidade PHP**. O objetivo foi garantir que o NotebookLM utilizasse materiais confiáveis, atualizados e complementares entre si.

Cada fonte foi escolhida para responder a uma necessidade específica durante o processo de aprendizagem.

| Fonte                                            | Objetivo da Fonte                                                                 | Contribuição para o Estudo                                                                                                            |
| ------------------------------------------------ | --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| PHPUnit – Documentação Oficial                   | Compreender os fundamentos dos testes automatizados em PHP.                       | Serviu como principal referência para conhecer a estrutura dos testes, assertions, casos de teste e boas práticas utilizando PHPUnit. |
| Laravel – Testing Documentation                  | Entender como o Laravel organiza os testes em aplicações modernas.                | Complementou o estudo mostrando como o framework simplifica a criação de testes unitários, de funcionalidades e de requisições HTTP.  |
| PHP: The Right Way                               | Estudar boas práticas de desenvolvimento em PHP.                                  | Auxiliou na compreensão de como escrever código mais organizado, facilitando sua manutenção e testabilidade.                          |

### Critérios utilizados na seleção das fontes

Durante a curadoria procurei utilizar materiais que atendessem aos seguintes critérios:

* **Oficialidade:** priorizando documentação oficial sempre que possível.
* **Atualização:** utilizando fontes mantidas ativamente pela comunidade.
* **Confiabilidade:** materiais amplamente utilizados por desenvolvedores PHP.
* **Complementaridade:** cada fonte aborda um aspecto diferente do tema, permitindo que o NotebookLM realize comparações e produza respostas mais completas.

Essa etapa foi importante para que o NotebookLM tivesse uma base de conhecimento consistente, aumentando a qualidade das respostas durante todo o processo de estudo.

## Como utilizei o NotebookLM

O NotebookLM foi utilizado como uma ferramenta de apoio ao estudo, permitindo centralizar as fontes de pesquisa, realizar perguntas baseadas nesses materiais e organizar o conhecimento adquirido durante o processo de aprendizagem.

Após selecionar as fontes, realizei o upload da documentação no NotebookLM e iniciei uma sequência de perguntas com o objetivo de compreender os conceitos fundamentais antes de avançar para assuntos mais específicos.

Durante o estudo, procurei evoluir os prompts de forma gradual. Em vez de solicitar apenas definições, busquei comparar conceitos, identificar diferenças entre abordagens e consolidar o conhecimento por meio de resumos e explicações baseadas exclusivamente nas fontes disponibilizadas.

Essa estratégia tornou o processo de aprendizagem mais organizado, pois todas as respostas permaneceram contextualizadas pelo material selecionado, reduzindo a necessidade de consultar diversas referências separadamente.

Ao longo da utilização do NotebookLM, observei que a qualidade das respostas estava diretamente relacionada à qualidade das fontes utilizadas e à clareza dos prompts elaborados. Perguntas mais específicas produziram respostas mais completas e úteis para o estudo.

O NotebookLM também foi utilizado para apoiar a criação do miniguia de estudo, do glossário de conceitos e da revisão dos principais tópicos aprendidos durante este projeto.

### Evolução dos Prompts

Durante a utilização do NotebookLM, percebi que perguntas muito genéricas resultavam em respostas igualmente genéricas. À medida que os prompts passaram a indicar claramente o objetivo da pergunta e a solicitar comparações, exemplos ou resumos, as respostas tornaram-se mais completas, organizadas e úteis para o estudo.

Essa experiência demonstrou que a qualidade dos resultados depende não apenas da ferramenta utilizada, mas também da forma como as perguntas são elaboradas.

## Dificuldades Encontradas (Cicatrizes)

Durante o desenvolvimento deste projeto, foi possível perceber que a qualidade do aprendizado utilizando o NotebookLM depende de alguns fatores importantes.

A primeira dificuldade foi definir um conjunto de fontes que fossem confiáveis, atualizadas e complementares. A seleção das referências teve impacto direto na qualidade das respostas produzidas pela ferramenta.

Outro desafio foi elaborar prompts suficientemente específicos. As primeiras perguntas tendiam a gerar respostas muito amplas. Com o refinamento dos prompts, passando a solicitar comparações, exemplos e resumos, as respostas tornaram-se mais objetivas e adequadas ao objetivo do estudo.

Também ficou evidente que o NotebookLM não substitui a leitura das fontes. A ferramenta auxilia na organização do conhecimento e na síntese das informações, mas a interpretação crítica do conteúdo continua sendo responsabilidade do estudante.

Como principal aprendizado, percebi que o uso de Inteligência Artificial produz melhores resultados quando existe planejamento do estudo, seleção criteriosa das fontes e elaboração consciente dos prompts.

## Miniguia de Estudo

Ao final do estudo realizado com o NotebookLM, foi possível consolidar os seguintes conceitos sobre testes de software aplicados ao desenvolvimento em PHP.

### 1. O que é Teste de Software?

Teste de software é o processo de verificar se uma aplicação apresenta o comportamento esperado, identificando falhas antes que o sistema seja disponibilizado aos usuários.

Além de aumentar a qualidade do software, os testes contribuem para reduzir riscos durante a manutenção e a evolução do código.

---

### 2. Testes em PHP utilizando PHPUnit

O PHPUnit é o framework de testes mais utilizado na comunidade PHP para criação de testes automatizados.

Durante o estudo, foram identificados alguns conceitos fundamentais:

* organização dos casos de teste;
* utilização de assertions para validar resultados;
* execução automatizada dos testes;
* isolamento das regras de negócio para facilitar a validação.

---

### 3. Testes em Laravel

O Laravel utiliza o PHPUnit como base, porém oferece recursos adicionais que simplificam a criação de testes.

Entre os principais recursos estudados destacam-se:

* Testes Unitários;
* Feature Tests;
* HTTP Tests;
* testes envolvendo banco de dados;
* factories e seeders para preparação dos dados de teste.

---

### 4. Boas práticas observadas

Durante o estudo, algumas boas práticas apareceram de forma recorrente nas fontes consultadas:

* escrever testes simples e objetivos;
* manter os testes organizados;
* utilizar dados de teste representativos;
* criar testes independentes entre si;
* executar os testes com frequência durante o desenvolvimento.

---

### 5. Principais aprendizados

Como resultado deste estudo, compreendi que o processo de testes não deve ser visto apenas como uma etapa de validação, mas como uma prática que contribui para a qualidade, manutenção e evolução do software.

Também percebi que o NotebookLM facilitou a consolidação do conteúdo ao reunir diferentes fontes de conhecimento em um único ambiente, permitindo comparar conceitos, produzir resumos e revisar os principais tópicos estudados.

## Glossário

| Termo               | Definição                                                                                                              |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Teste de Software   | Processo utilizado para verificar se um software apresenta o comportamento esperado e atende aos requisitos definidos. |
| PHPUnit             | Framework de testes automatizados para aplicações desenvolvidas em PHP.                                                |
| Teste Unitário      | Teste que valida o funcionamento de uma pequena unidade de código, como uma função ou método.                          |
| Teste de Integração | Teste que verifica se diferentes componentes do sistema funcionam corretamente em conjunto.                            |
| Feature Test        | Tipo de teste utilizado no Laravel para validar o funcionamento de uma funcionalidade completa da aplicação.           |
| Assertion           | Instrução utilizada em um teste para verificar se o resultado obtido corresponde ao resultado esperado.                |
| Testabilidade       | Característica de um software que facilita a criação e execução de testes.                                             |
| Cobertura de Testes | Indicador que demonstra quais partes do código são exercitadas pelos testes automatizados.                             |
| Factory             | Recurso do Laravel utilizado para gerar dados de teste automaticamente.                                                |
| Seeder              | Recurso utilizado para popular o banco de dados com informações necessárias durante os testes.                         |
